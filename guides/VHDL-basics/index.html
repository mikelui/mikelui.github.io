Time in VHDL

Before we move on, we have to distinguish one more difference between between traditional programming and HDLs. Traditional programming languages are based on the idea of computation sequences; in other words, steps are executed one after the other in the order they appear. For example, imperative languages guarantee that each statement happens sequentially, so we can set data in one step and read it back afterwards.

# pseudo-code
a = 5
a.increment()
a  # a == 6

In VHDL, we model a circuit, so data just propagates through wires.

ENTITY pass_through IS
    PORT ( a    : IN BIT; 
           b, c : OUT BIT);
END pass_through;

ARCHITECTURE df OF pass_through IS
    SIGNAL between : BIT;
BEGIN
    -- b and between are set
    -- at the same time!
    b <= between;
    between <= a;
    c <= between
END df;
